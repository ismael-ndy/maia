SOFTWARE REQUIREMENTS SPECIFICATION (SRS)
Project Name: MAIA
Version: 1.0
Type: Web Application (Frontend + Backend API)

==================================================

INTRODUCTION
==================================================

1.1 Purpose
This document specifies the requirements for Maia, a web-based AI-powered mental health support platform.
The goal is to guide frontend development and ensure correct integration with the existing backend API.

This SRS is written specifically for frontend generation tools (v0, Figma AI, etc.).

1.2 Scope
Maia provides:

A patient-facing AI chat interface

A therapist-facing dashboard

Secure role-based access

AI-assisted safety monitoring and reporting

Maia does NOT:

Diagnose conditions

Replace professional therapy

Provide medical advice

==================================================

3. USER CHARACTERISTICS

3.1 Patients

Non-technical users

Emotionally vulnerable at times

Need calm, simple UI

Patient abilities:

Chat with Maia (AI assistant)

View chat history

View assigned therapist info

Accept or deny therapist assignment requests

3.2 Therapists

Professional users

Manage multiple patients

Therapist abilities:

Request assignment to patients via email

View assigned patients

Receive alerts

Generate and view weekly reports

==================================================
4. FUNCTIONAL REQUIREMENTS
4.1 AUTHENTICATION

Login
Endpoint: POST /auth/token
Inputs:

Email

Password

Outputs:

JWT access token

Frontend behavior:

Store token securely

Attach token to all protected requests

Redirect based on user role

Signup
Endpoint: POST /auth/signup
Roles:

Patient

Therapist

4.2 PATIENT CHAT

Send Message (Streaming)
Endpoint: POST /chats/messages/stream
Protocol: Server-Sent Events (SSE)

Features:

Streaming AI responses

Typing indicator

Auto-scroll

Timestamped messages

Chat History
Endpoint: GET /chats/messages

Features:

Loaded on chat open

Ordered oldest → newest

Read-only for therapists

4.3 THERAPIST–PATIENT ASSIGNMENT

Send Assignment Request
Endpoint: POST /friend-requests/?patient_email=...

View Requests
Endpoint: GET /friend-requests

Accept Assignment
Endpoint: POST /friend-requests/{therapist_id}/accept

Patients can accept or deny requests.

4.4 THERAPIST DASHBOARD

List Patients
Endpoint: GET /therapists/patients

Display:

Patient name

Email

Clicking a patient opens patient detail view.

4.5 PATIENT DETAIL VIEW (THERAPIST)

Get Patient Info
Endpoint: GET /therapists/patients/{patient_id}

Tabs:

Weekly Reports

Alerts

4.6 REPORTS

Generate Weekly Report
Endpoint: POST /therapists/patients/{patient_id}/reports

Frontend behavior:

Button-triggered

Disable button while generating

List Reports
Endpoint: GET /therapists/patients/{patient_id}/reports

View Report
Endpoint: GET /therapists/patients/{patient_id}/reports/{report_id}

Reports are read-only and structured.

4.7 ALERTS (PLANNED)

Alerts Tab (Therapist Dashboard)
Endpoint: GET /alerts

Displays:

Patient name

Risk level

Urgency

Timestamp

Alerts do NOT show raw chat content.
